**理性决策Agent（SMA+OUT）**：

```python
class RationalAgent:
    """
    理性决策基线Agent

    决策流程:
        1. SMA预测: D̂_t^L = (L/p) * Σ D_{t-i}
        2. 预测误差: ê_t^L = C_{L,ρ} * (σ_ε/p) * sqrt(Σ e_{t-i}²)
        3. 期望库存: S_t = D̂_t^L + z * ê_t^L
        4. 订货: q_t = S_t - (NS_t + WIP_t)
    """

    def decide(self, k, NS, WIP, current_demand):
        """为节点k做理性决策，返回订货量 q_t"""
        if k not in self.demand_history:
            self.init_node(k)

        # 记录需求
        self.demand_history[k].append(current_demand)

        # 更新预测误差
        if self.last_forecast.get(k) is not None:
            e_t = current_demand - self.last_forecast[k]
            self.error_history[k].append(e_t)

        # 预测
        forecast_L, error_std = self.predict(k)

        if forecast_L is None:
            # 数据不足，使用需求本身
            q_t = max(0, current_demand)
        else:
            # 期望库存 S_t = D̂_t^L + z * ê_t^L
            S_t = forecast_L + self.z * error_std
            # 订货 q_t = S_t - (NS + WIP)
            q_t = S_t - (NS + WIP)
            q_t = max(0, q_t)  # 不允许负订货

        return q_t
```

**AR(1)需求生成**：

```python
def _generate_demand(self):
    """生成AR(1)需求: D_t = d + ρ·D_{t-1} + ε_t"""
    if len(self.customer_demand_history) == 0:
        D_t = self.d
    else:
        D_prev = self.customer_demand_history[-1]
        noise = self.rng.normal(0, self.sigma_eps)
        D_t = self.d + self.rho * D_prev + noise
    return max(0, D_t)  # 需求非负
```

**DQN值网络（纯NumPy实现）**：

```python
class QNetwork:
    """
    DQN值网络 (纯NumPy实现)
    网络结构: state_dim(5) -> 64 -> 64 -> action_dim(30) (ReLU激活)
    使用He初始化和Adam优化器
    """

    def forward(self, x):
        self.x = x
        self.z1 = x @ self.W1 + self.b1
        self.a1 = np.maximum(0, self.z1)  # ReLU
        self.z2 = self.a1 @ self.W2 + self.b2
        self.a2 = np.maximum(0, self.z2)  # ReLU
        self.z3 = self.a2 @ self.W3 + self.b3
        return self.z3  # Q值

    def backward(self, grad_q, batch_size):
        # 标准反向传播，计算各层参数梯度
        grad_W3 = self.a2.T @ grad_q / batch_size
        grad_b3 = np.sum(grad_q, axis=0) / batch_size
        # ... (省略中间层)
        return {'W1': grad_W1, 'b1': grad_b1,
                'W2': grad_W2, 'b2': grad_b2,
                'W3': grad_W3, 'b3': grad_b3}
```

**IDMR状态空间（5维）**：

```python
def get_state(self, env, k=3):
    """
    构建状态向量 (论文公式: 5维)
    s_t = [S_{t-1}^3, WIP_{t-1}^3, q_{t-1}^2, Trans_{t-2}^3, q_{t-1}^3]

    状态归一化: 各维度除以缩放因子
    """
    node = env.nodes[k]
    s_stock = node.net_stock / 100.0              # 分销商上期库存
    wip = sum(node.pipeline) / 100.0              # 分销商在途库存
    q_downstream = upstream_node.order_placed / 50.0  # 批发商上期订单
    trans = node.pipeline[0] / 50.0               # 分销商前期运输货物
    q_self = node.order_placed / 50.0             # 分销商上期订货量
    return np.array([s_stock, wip, q_downstream, trans, q_self])
```

**ε-greedy动作选择 + 惩罚机制**：

```python
def select_action(self, state, force_zero=False):
    """
    ε-greedy动作选择
    force_zero: 惩罚机制触发时，强制a=0（禁止订货）
    """
    if force_zero:
        return 0  # 惩罚: 禁止订货

    if random.random() < self.epsilon:
        # 探索: 在[11,40]内随机
        return random.randint(self.action_min, self.action_max)
    else:
        # 利用: 选择Q值最大的动作
        q_values = self.q_net.forward(state.reshape(1, -1))
        action_idx = np.argmax(q_values)
        return self.action_min + action_idx  # 映射到[11, 40]
```

```python
def _compute_classical_avg_inventory(self, d, rho, sigma_eps, L, p, z,
                                      C_L_rho, initial_inventory, K, seed):
    """
    运行经典供应链Baseline, 计算分销商(k=3)的平均库存

    李勇论文惩罚机制: "当积压库存达到相同需求下经典多级供应链
    对应节点的平均库存时, 禁止向上游订货"
    → 惩罚阈值 = 经典供应链分销商的实际平均库存
    """
    from supply_chain_env import SupplyChainEnv, RationalAgent
    env = SupplyChainEnv(
        d=d, rho=rho, sigma_eps=sigma_eps, L=L, p=p, z=z,
        C_L_rho=C_L_rho, initial_inventory=initial_inventory, K=K,
        seed=seed,
    )
    agent = RationalAgent(L=L, p=p, z=z, C_L_rho=C_L_rho, sigma_eps=sigma_eps)
    for k in range(1, K + 1):
        agent.init_node(k)

    env.reset()
    ns_history_3 = []
    warmup = 2000  # 预热期

    for t in range(warmup):
        D_t = env._generate_demand()
        env.customer_demand_history.append(D_t)
        downstream_demand = {1: D_t}

        for k in range(1, env.K + 1):
            node = env.nodes[k]
            demand_k = downstream_demand.get(k, 0)
            arrived = node.pipeline[0] if len(node.pipeline) > 0 else 0.0
            node.net_stock += arrived
            if len(node.pipeline) > 0:
                node.pipeline.popleft()

            q_t = agent.decide(k, node.net_stock, sum(node.pipeline), demand_k)
            q_t = max(0, q_t)
            node.order_placed = q_t
            node.order_history.append(q_t)
            downstream_demand[k + 1] = q_t

            fulfilled = min(max(node.net_stock, 0), demand_k)
            node.net_stock -= fulfilled
            node.demand_history.append(demand_k)
            node.pipeline.append(q_t)

            if k == 3:
                ns_history_3.append(node.net_stock)

    avg_inv = float(np.mean(ns_history_3)) if ns_history_3 else initial_inventory
    print(f"  [惩罚机制] 经典供应链分销商平均库存={avg_inv:.2f} (预热{warmup}步)")
    return avg_inv  # 实测值 ≈ 352.51
```

## 5 关键修复记录

### 5.1 问题描述

**用户反馈**：根据李勇论文，智慧决策中分销商的服务水平应当提高，但我们复现的结果却是降低的（SL从99.76%下降到88.88%）。

### 5.2 问题1：inventory_bonus正向激励干扰

**现象**：奖励函数中`inventory_bonus`权重(0.3)超过了缺货惩罚损失(0.11)，导致DQN学到了"少订货以维持低库存反而获得更高奖励"的错误策略。

**根因分析**：
- `inventory_bonus`设计初衷是对冲"损失厌恶"导致的过度订货
- 但在实际运行中，DQN发现不订货→库存低→精准匹配→获得bonus > 缺货惩罚
- 导致DQN学到"少订货"策略，造成严重缺货

**修复方案**：在`batch_runner.py`中关闭`inventory_bonus`

```python
# batch_runner.py 第185行
cfg.idmr.enable_inventory_bonus = False    # 关闭正向激励 (修复SL下降问题)
```

**修复效果**：SL从88.88%有所提升，但仍未达到目标，说明inventory_bonus不是唯一问题。

### 5.3 问题2：惩罚阈值计算错误（根因）

**现象**：关闭`inventory_bonus`后，分销商SL仍然是88.88%，说明根因在别处。

**诊断过程**：

编写诊断脚本`diagnose_penalty.py`，检查惩罚机制触发情况：

```python
# diagnose_penalty.py 关键输出
分销商实际平均库存: 316.95
惩罚阈值 (initial_inventory × penalty_threshold): 10 × 5.0 = 50
惩罚触发比例: 98.0%  ← 98%的周期IDMR被禁止订货！
```

**根因定位**：

`idmr_agent.py`第505行原代码：
```python
self.classical_avg_inventory = {3: initial_inventory}  # = {3: 10}
```

李勇论文惩罚机制定义："当积压库存达到相同需求下经典多级供应链对应节点的平均库存时，禁止向上游订货"。

这里的"平均库存"应该是**经典供应链分销商的实际平均库存**，而代码错误地使用了`initial_inventory=10`。

实际经典供应链分销商平均库存 ≈ 316.95（2000步预热测得），远大于10。

**后果**：
- 惩罚阈值 = 10 × 5.0 = 50
- 实际分销商库存 ≈ 316.95 >> 50
- 98%的周期触发惩罚，IDMR被强制禁止订货（`force_zero=True`）
- IDMR无法补货 → 严重缺货 → SL崩塌至88.88%

**修复方案**：

1. 在`idmr_agent.py`中新增`_compute_classical_avg_inventory`方法，运行2000步Baseline获取实际平均库存（352.51）
2. 在`batch_runner.py`中设置`penalty_threshold = 1.0`（达到经典平均库存时禁止订货）

**修复代码**：

```python
# idmr_agent.py 第504-561行 (新增方法)
self.classical_avg_inventory = {3: self._compute_classical_avg_inventory(
    d, rho, sigma_eps, L, p, z, C_L_rho, initial_inventory, K, seed)}

# batch_runner.py 第186行
cfg.idmr.penalty_threshold = 1.0  # 修复: 达到经典平均库存时禁止订货
```

|:---|:---:|:---:|:---:|
