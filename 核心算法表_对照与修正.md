# 核心算法表对照与修正

## 判定：不完全一致，存在7处关键差异

用户提供的算法表与实验代码（`marl_supply_chain_env.py`、`emotion_module.py`、`dynamic_events.py`、`run_exp2_20k.py`）逐行对照后，发现以下差异：

### 差异对照表

| 行号 | 用户伪代码 | 实际代码 | 差异类型 |
|:----:|:-----------|:---------|:---------|

## 修正后的核心算法表

以下为基于实验代码修正后的算法表，格式与用户原文一致：

```
算法1：多智能体情绪感知人智协同决策系统（MAEA-HCDS）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
输入：仿真周期T=20000，需求参数(d=10, ρ=0.5, σ_ε=5)，提前期L=2
      情绪参数(α=0.7, γ=2.0, w_s=1.0, w_m=0.5, w_e=0.3)
      传染参数(θ=0.3, p_c=0.3, s_c=0.4)
      动态事件概率(p_d=0.00265, p_s=0.00115)
      成本参数(h=1.0, b=2.0)，观测维度=8
输出：各节点订货序列{q_t^k}，情绪序列{E_t^k}，绩效指标(BWE, SL, Cost)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 1. 初始化MARLSupplyChainEnv(PettingZoo AECEnv)，创建4个SupplyChainAgent
    （零售商k=1 → 批发商k=2 → 分销商k=3 → 制造商k=4）
 2. 为每个Agent独立挂载EmotionState(α, γ, w_s, w_m, w_e)
 3. 初始化CommunicationChannel(拓扑=下游→上游，共享模式=forecast+inventory)
 4. 初始化DynamicEventTrigger(p_d, p_s, θ, p_c, s_c, seed=42)
 5. For t = 1 : T do
 6.   // —— 动态事件触发 ——
 7.   触发需求突变(m∈{2.0, 0.5})与维护供应中断状态(持续3-5周期)
 8.   生成顾客需求 D_t = d + ρ·D_{t-1} + ε_t；若需求突变则 D_t ← D_t × m
 9.   For k = 1 : 4 do  // AECEnv顺序决策：下游先决策，订单逐级上传
10.     // —— 库存更新与履约 ——
11.     收到上游L期前到货（k=4制造商：断供期间到货=0）
12.     履约下游需求：fulfilled = min(NS, D_t^k)，stockout = max(0, D_t^k - fulfilled)
13.     // —— 行为感知：情绪演化（式1-2）——
14.     Φ_t = -w_s·stockout_rate + w_m·match_factor - w_e·excess_rate
15.     E_t^k = tanh(α·E_{t-1}^k + γ·Φ_t)
16.     // —— 情绪传染（式8，周期末由DynamicEventTrigger执行）——
17.     If stockout_rate > θ AND random() < p_c:
18.       E_t^{k+1} ← tanh(E_t^{k+1} + (-s_c))  // 固定恐慌冲击注入上游
19.     // —— 人机协同机制1：传授经验（理性决策作为基础策略）——
20.     q_base = SMA(D_{t-p:t-1}) + z·σ_L·√(L+1)  // OUT策略
21.     // —— 情绪调节订货量（式5-6）——
22.     If E_t^k < 0: q_t^k = q_base × (1.0 + 0.5|E_t^k|)  // 恐慌放大
23.     Elif E_t^k > 0: q_t^k = D_t^k + (q_base - D_t^k)×(1.0 - 0.3·E_t^k)  // 乐观缩减
24.     // —— 情绪调节奖励权重（式3-4）——
25.     w_s_eff = w_s^0·(1 + |E_t^k|)  // 恐慌放大缺货惩罚
26.     w_b_eff = w_b^0·(1 + E_t^k)    // 乐观放大正向激励
27.     // —— 人机协同机制2：正向激励目标重塑（式7）——
28.     c_r = NS_t^k / forecast  // 库存覆盖率
29.     If c_r ∈ [0.8, 1.5]: match_factor = max(0, 1-|c_r-1|), bonus = w_b_eff·match_factor
30.     // —— 综合奖励（式7）——
31.     r_t^k = fill_rate + bonus - w_s_eff·stockout_rate - w_h·max(0, NS)
32.     // —— 人机协同机制3：协同通信（信息共享）——
33.     广播{forecast, inventory, emotion_label}至通信通道
34.     上游k+1从通信通道接收下游k的forecast与inventory（观测第6-7维）
35.     // —— 构建局部观测（8维）——
36.     s_t^k = [NS, WIP, D_down, arrival, D_last, E_t, forecast_shared, inventory_shared]
37.   End for
38. End for
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
注1：在Exp_1/Exp_1b中，分销商(k=3)使用ContinualIDMRAgent(DQN+PER+EWC)替代
    理性决策，第20-23行替换为ε-greedy动作选择→情绪调节→PER优先级采样→
    DQN梯度更新→EWC正则约束，每target_update_freq步更新目标网络。
注2：情绪感知噪声 E_perceived = clip(E_true + N(0, σ_noise), -1, 1)（式15），
    σ_noise=0.15时应用于Q网络输入第6维，真实情绪仍用于环境动力学。
```

## 修正要点说明

### 1. 情绪传染公式修正（第17-18行）

| 项目 | 用户原版 | 修正版 |
|:-----|:---------|:-------|
| 触发条件 | 无条件概率η | `stockout_rate > θ(=0.3)` AND `random() < p_c(=0.3)` |
| 传染方式 | 线性插值 `E + η(E_{k-1} - E)` | 固定冲击 `tanh(E + (-s_c))`，s_c=0.4 |
| 传染方向 | 未明确 | 下游k → 上游k+1（单向） |
| 代码位置 | — | `dynamic_events.py` L200-278 |

### 2. 决策机制修正（第20-23行）

| 项目 | 用户原版 | 修正版 |
|:-----|:---------|:-------|
| 决策方式 | DQN ε-greedy | RationalAgent(SMA+OUT) + 情绪调节 |
| 适用实验 | 暗示Exp_2使用DQN | Exp_2使用理性决策；Exp_1/1b使用DQN |
| 代码位置 | — | `run_exp2_20k.py` L140-150 |

### 3. 奖励函数修正（第31行）

| 项目 | 用户原版 | 修正版 |
|:-----|:---------|:-------|
| 奖励公式 | `fill_rate + bonus - holding_penalty` | `fill_rate + bonus - w_s_eff·stockout_rate - w_h·max(0,NS)` |
| 缺失项 | 缺货惩罚 | 补充情绪调节的缺货惩罚项 |
| 代码位置 | — | `marl_supply_chain_env.py` L435-506 |

### 4. 协同通信内容修正（第33-34行）

| 项目 | 用户原版 | 修正版 |
|:-----|:---------|:-------|
| 传递内容 | q_t（订货量）与 E_t（情绪值） | forecast（预测需求）、inventory（库存）、emotion_label（情绪标签） |
| 代码位置 | — | `marl_supply_chain_env.py` L556-573 |

### 5. 新增动态事件触发（第6-8行）

用户原版完全遗漏，实际代码中76次动态事件是Exp_2的核心特征：
- 需求突变：53次，D_t × {2.0, 0.5}
- 供应中断：23次，持续3-5周期，制造商到货=0
- 情绪传染：2830次，30%概率上游传染
- 代码位置：`dynamic_events.py` L110-165, L200-278

### 6. 新增8维局部观测（第36行）

用户原版未描述观测构建，实际代码中观测为8维向量：
```
[NS, WIP, D_downstream, arrival, D_last, E_t, forecast_shared, inventory_shared]
```
其中第6-7维来自协同通信通道，代码位置：`marl_supply_chain_env.py` L380-433

### 7. 惩罚机制修正

用户原版第17行 `If NS > 5×NS_avg: a=0` 在`MARLSupplyChainEnv`中未实现。
实际替代机制为奖励函数中的 `holding_penalty = w_h·max(0, NS)`，w_h=0.0001。
原惩罚机制仅在单智能体`IDMRSupplyChainEnv`中实现。
